## API Reference

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `variant` | `'default' \| 'secondary' \| 'destructive' \| 'outline' \| 'ghost' \| 'link'` | `'default'` | Visual style variant |
| `size` | `'sm' \| 'default' \| 'lg' \| 'icon'` | `'default'` | Button size |
| `disabled` | `boolean` | `false` | Disable interaction |
| `asChild` | `boolean` | `false` | Render as child element |
| `type` | `'button' \| 'submit' \| 'reset'` | `'button'` | Button type |
| `onClick` | `(event: React.MouseEvent) => void` | `undefined` | Click handler |
| `className` | `string` | `undefined` | Additional CSS classes |

## Accessibility

- Keyboard navigation with Tab, Space, and Enter
- Proper ARIA attributes for screen readers
- Clear focus indicators
- Loading states announced to assistive technology

## Best Practices

### Do's ✅
- Use primary buttons for main actions
- Keep button text concise and action-oriented
- Provide loading states for async operations
- Use appropriate variants for different contexts

### Don'ts ❌
- Don't use multiple primary buttons in same context
- Avoid overly long button text
- Don't nest interactive elements inside buttons
- Don't use disabled buttons without clear reason
